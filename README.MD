  ITAD only logs *price changes*, not continuous snapshots. At first, this made analysis confusing (many â€œ0% discountâ€ rows).  
  I solved this by treating the dataset as an **event log** and then engineering per-game summaries like â€œmax discountâ€ and â€œtime to first sale.â€

- **Skewed variables**  
  Review counts ranged from 1 to 2.5M, hiding most of the distribution. I used log-scaling to reveal long-tail patterns and interpret them properly.
